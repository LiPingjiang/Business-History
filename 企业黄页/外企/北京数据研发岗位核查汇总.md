# 北京外企数据研发岗位核查汇总

> 《企业史》企业黄页 — 附属核查记录

**核查范围：** 企业黄页收录的全部290家外资企业

**核查方式：** 使用浏览器自动化工具逐一打开各公司官方招聘页面，搜索/筛选北京（Beijing）地点的在招职位，重点关注数据研发相关岗位（数据工程师、软件研发、数据科学家、后端开发、AI/大数据等技术研发类岗位，不含销售、市场、HR等非技术岗位）

**核查时间：** 2026年7月

---

## 一、总体结论

本次对290家外资企业官方招聘页面逐一进行了核查尝试。核查过程中发现一个贯穿始终的系统性问题：**当前网络环境对约80%~95%的公司官网域名返回连接被拒绝错误（`ERR_CONNECTION_CLOSED`）**，多次重试均未能改善，而Google、百度等主流网站可正常访问，说明这是针对特定域名/CDN的网络限制，并非这些公司官网本身存在问题。因此，本次核查结果应理解为"当前网络条件下的核查结果"，而非"这些公司北京没有招聘需求"的结论。

在能够成功访问并完成搜索的约30余家公司中，**确认发现北京地区数据研发相关在招岗位的公司共6家**：

| 公司 | 数据研发岗位数量（约） | 代表性岗位 |
|------|----------------------|-----------|
| 亚马逊（Amazon） | 约23个技术类岗位 | Software Development、Business Intelligence、Solutions Architect、Applied Science等 |
| 英伟达（NVIDIA） | 多个 | Senior Software Engineer (Networking/RDMA)、Senior Autonomous Driving Software Engineer、Deep Learning Solution Architect等 |
| 谷歌（Google） | 3个左右 | Software Engineer (gReach Program)、Strategic Cloud Architect、Forward Deployed Architect Manager (GenAI) |
| 微软（Microsoft） | 至少1个 | Senior Applied Scientist - CoreAI Voice Agent |
| 英特尔（Intel） | 1个（主地点上海，含北京） | Senior Gen AI Software Solutions Engineer |
| Airbnb | 2个 | AI Engineer, Community Support Engineering（远程）等 |

其余能访问但未发现数据研发岗位的公司：MongoDB（北京3个岗位均为销售类）、Elastic（无在招职位）、Datadog（无匹配职位）、SAP（北京仅有实习/销售岗）、麦当劳中国、联邦快递（搜索Beijing结果为0）、Zoom（无中国选项）、甲骨文（北京无明确数据研发岗）、IHG洲际酒店（无北京岗位）、开市客等。

McKinsey（麦肯锡）有大量Data Engineer/AI Engineer岗位（QuantumBlack团队），但沙盒环境下未能确认这些岗位是否落地北京，需要后续复核。

## 三、方法论说明与局限性

本次核查覆盖了黄页收录的全部290家外资企业，但受限于当前沙盒网络环境对大量海外域名的连接限制，实际能够完成"搜索北京岗位"这一步骤的公司不足总数的15%。对于返回`ERR_CONNECTION_CLOSED`的公司，不能得出"该公司在北京没有数据研发岗位"的结论，只能标注为"当前网络环境下无法核实"。

对于页面可访问但职位列表为单页应用（SPA）无法渲染、或依赖需要鉴权的API接口的情况（如Adobe、Snowflake、万事达卡、思科、高通、苹果等），同样无法得出实质性结论。

## 四、后续建议

若需要更完整、更可靠的核查结果，建议：

在无网络限制的环境下重新执行本次核查流程，尤其是欧洲（德国、法国、荷兰、瑞典、丹麦、芬兰、意大利）、日本、韩国、英国、新加坡、加拿大等区域，这些区域几乎全军覆没式地遭遇连接失败，很可能只是网络出口问题。

对于SPA/API鉴权失败的公司（Adobe、Snowflake、思科、高通、苹果、耐克、高盛等），可尝试通过公开的职位搜索API或直接搜索引擎缓存页面获取数据。

通过第三方招聘平台（BOSS直聘、猎聘、LinkedIn中国版）交叉核实这些公司在北京的招聘信息，尤其是麦肯锡（QuantumBlack数据团队）等已知有相关业务但未能确认地点归属的公司。

对已确认发现北京数据研发岗位的6家公司（亚马逊、英伟达、谷歌、微软、英特尔、Airbnb），建议定期复查这些岗位的最新状态，因为招聘信息更新频繁。
